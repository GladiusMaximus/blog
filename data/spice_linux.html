I have to use spice for Digital Electronics class. The class is based on people using coolspice. That is a windows only product. My windows is too slow to be usable. Here is my guide on how to use spice on debian/ubuntu

First, install geda and ngspice from the debian repo. 

sudo apt-get install geda ngspice

Now let's start using it. Open up gschem through any application launcher. Look around. This program is heavily based on keyboard shortcuts. You can get to everything through the menu-bar at the top or the context-menu, but the keyboard is much more efficient. Press I to insert components, F S to save.

To insert a power-source, use vdc-1.sym. This inserts a spice-specific power-source. You need to insert a ground gnd-1.sym as well. Here is an example.

You should give every component a unique name. Double-click the component (or press E E if the component is selected), and change the refdes field (keep the first letter the same, because the first letter indicates what component it is to spice). Add a value field to describe the device. Consult this chart for whot to put in the value field. To name a wire, add a netname.

Now for spice, you may add spice directives one-by-one with spice-directive-1.sym. What I normally do is have all of my spice directives in a file and include the file with spice-include-1.sym. Mine usually defines any external models I am using, defines a simulation, and a creates control section that executes during the simulation. Here is a sample. 

.include ./MOSIS.txt
.tran 2p 5n
.options TEMP=125
.save v(Vout)
.save v(Vin)
.control
run 
plot vin vout
meas tran tperiod TRIG V(Vin)  val=0.5  rise=1 targ V(Vin)  val=0.5  rise=2
meas tran trise   TRIG V(Vout) val=0.18 rise=1 targ V(Vout) val=1.62 rise=1
meas tran tfall   TRIG V(Vout) val=1.62 fall=1 targ V(Vout) val=0.18 fall=1
meas tran tphl    TRIG V(Vin)  val=0.9  rise=2 targ V(Vout) val=0.9 fall=2
meas tran tplh    TRIG V(Vin)  val=0.9  fall=1 targ V(Vout) val=0.9 rise=1
.endc

Once you have built your circuit, you need to compile it to a spice net-list. Go into terminal and do. 

gnetlist -g spice-sdb -O sort_mode -o OUTPUT.net INPUT.sch
cat OUTPUT.net
If you know how to read spice netlists, then make sure this is valid before continuing.

Then, launch ngspice with your the netlist as input. ngspice should run whatever simulation you defined. 

ngspice OUTPUT.net
